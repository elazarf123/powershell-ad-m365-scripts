# Security Policy

## ⚠️ No Secrets in This Repository

This repository contains **PowerShell automation scripts** for Microsoft 365 and Active Directory administration. These scripts interact with sensitive infrastructure. The following security policy governs all contributions.

---

## Absolute Rules

| Rule | Detail |
|------|--------|
| **No credentials in code** | Passwords, API keys, client secrets, PFX passphrases, and tokens must **never** be committed |
| **No tenant data** | UPNs, ObjectIDs, tenant IDs, group names, and domain names belonging to real environments must not appear in code or sample files |
| **No certificate private keys** | `.pfx`, `.pem`, `.key` files are blocked by `.gitignore` and must not be force-added |
| **No connection strings** | Hard-coded SQL, storage, or Graph connection strings are prohibited |

If you accidentally commit a secret, **immediately rotate it** — deleting the file is not enough because git history retains the value. Then open an issue so the commit history can be purged.

---

## Recommended Secret-Free Authentication Patterns

### Interactive / Developer Use
```powershell
# Delegates auth to the user — no secret stored anywhere
Connect-MgGraph -Scopes "User.Read.All","Reports.Read.All"
```

### Unattended / CI Automation (Certificate-Based — Preferred)
```powershell
# App Registration with certificate — private key stays in Key Vault or local cert store
Connect-MgGraph -ClientId $env:APP_CLIENT_ID `
                -TenantId $env:TENANT_ID `
                -CertificateThumbprint $env:CERT_THUMBPRINT
```
> Store `APP_CLIENT_ID`, `TENANT_ID`, and `CERT_THUMBPRINT` in environment variables,  
> GitHub Actions secrets, Azure Key Vault, or a managed identity — **not in the script file**.

### Managed Identity (Azure-hosted workloads)
```powershell
Connect-MgGraph -Identity   # Uses the VM/Function App's system-assigned managed identity
```

### Using Azure Key Vault to retrieve a secret at runtime
```powershell
$secret = Get-AzKeyVaultSecret -VaultName "MyVault" -Name "GraphClientSecret" -AsPlainText
$secureSecret = ConvertTo-SecureString $secret -AsPlainText -Force
$credential  = [System.Management.Automation.PSCredential]::new($env:APP_CLIENT_ID, $secureSecret)
Connect-MgGraph -ClientSecretCredential $credential -TenantId $env:TENANT_ID
```

## Least-Privilege Graph API Permissions

Each script documents its required Graph scopes in the `.NOTES` comment block. Request **only** the permissions listed — do not grant broader scopes for convenience.

| Script | Minimum Required Scopes |
|--------|------------------------|
| `Get-M365LicenseReport.ps1` | `User.Read.All`, `Organization.Read.All` |
| `Get-LicenseOptimizationReport.ps1` | `User.Read.All`, `Organization.Read.All`, `AuditLog.Read.All` |
| `Get-StaleGuestReport.ps1` | `User.Read.All`, `AuditLog.Read.All` |
| `Export-ConditionalAccessPolicies.ps1` | `Policy.Read.All` |
| `Get-IntuneDeviceCompliance.ps1` | `DeviceManagementManagedDevices.Read.All`, `User.Read.All` |

---

## Secure Storage Checklist Before Running Scripts

- [ ] Credentials are stored in environment variables, a secrets manager, or a certificate store — **not in the script**
- [ ] The service account / app registration has only the permissions listed above
- [ ] Output CSV files are saved to an access-controlled location (not a shared drive or public storage)
- [ ] Logs generated by scripts do not contain passwords, tokens, or full certificate data
- [ ] The repo's `.gitignore` is in place so output files, `.pfx`, and `.env` files cannot be accidentally staged

---

## Reporting a Security Issue

If you discover a security vulnerability in these scripts (e.g., an injection risk, an insecure default, or an accidental secret in history):

1. **Do not** open a public GitHub Issue.
2. Email the maintainer directly: see the [About section in README.md](./README.md#about).
3. Include a description, the affected script, and reproduction steps.

Maintainers aim to acknowledge security reports within 48 hours and publish a fix within 7 days.
